# AWS Stroage Service
---
## 1. Amazon EBS(Elastic Block Store)
---
### EC2와 함께 사용할 수 있는 확장 가능한 고성능 블록 스토리지
-  Amazon EBS 볼륨 - Amazon EC2 인스턴스 연결되는 스토리지 볼륨 해당 볼륨은 컴퓨터에 연결되는 로컬 하드 드라이브 처럼 사용가능 
- EBS 를 다른 EC2에 부착은 가능하나 동시에는 사용할 수 없음 (특수 설정 필요)
-  Amazon EBS 스냅샷 - Amazon EBS 볼륨을 데이터를 백업하는 스냅샷을 생성 가능
- 스냅샷 은 이미지 처럼 언제든지 새로운 볼륨으로 복원 하는 데 가능
### 기능 
#### 1.  여러 가지 볼륨 유형 
- 빠른 속도와 트랜잭션 워크로드를 위한 SDD
- 처리량 집얍적 워크로들 위한 HDD 
#### 2. 확장성
- 필요선이 충족되는 용량 및 성능 사양으로 볼륨 생성 가능
- 용량을 늘리거나 성능 조정이 가능하다.
#### 3. 데이터 보호
- Amazon EBS 암호화 사용 EBS 볼륨과 EBS 스냅샷을 암호화가 가능
- 암호화 작업은 보안을 모두 보장하기 위해 Amazon Ec2 인스턴스를 호스팅하는 서버에서 이루어짐
#### 4. 데이터 보관
- EBS 스냅샷 아카이브에는 규제 및 구정 준수 또는 프로젝트 릴리르르 위해 90일 이상 유지해야 하는 특정 시점의 전체 EBS 스냅샷 복사본을 보관하는 서비스 제공

### Amazon EBS 최적화
- 최적화 인스턴스는 Amazon EBS I/O를 전용 용량을 추가로 제공  -> AMAzon EBS I/O 성능을 극대화 
- EBS 최적롸 인스턴스와 같이 이용시 전용 대역폭을 제공
- 처리량이 매우 높은 환경에서 용이

## 2. Amazon S3 (Simple Storage Service)
---
### 업계 최고의 확장성, 데이터 가용성 , 보안 및 성능을 제공하는 객체 스토리지 서비스
-  다양한 사용 사례에서 원하는 양의 데이터를 저장하고 보호 가능
- 스토리지 클래스 구별을 통한 비용 최적화 가능 (S3 Standard -> S3 IA -> S3 Glacier Flexible -> S3 Glacier Deep Archive)
- 엑세스 패턴이 변경되거나 알 수없는 패턴이 있는 데이터는 S3 Intelligent-Tiering에 저장하여 자동 관리 
### 스토리지 관리
- **S3 life cyle(수명주기)**- 수명 주기 구성으로 객체를 관리하고 수명 주기 동안 비용 효율적으로 저장 객체를 다른 S3 스토리지 클래스로 전환 하거나 수명이 다한 객체를 만료
-  **S3 객체 잠금** - 고정된 시간 동안 또는 무기한으로 객체의 삭제 또는 덮어쓰기를 방지  데이터 보호 계층을 추가하는 도움
- **S3 복제** - 대기 시간 단축, 규정준수, 보안 및 기타 사용 사례를 위한 객체. 객체의 각 메타데이터, 객체 태그를 동일하거나 하나의 이상의 버킷에 복제
- **S3 배치 작업** - Amazon S3 콘솔에서 단일 S3 API 요청이나 클릭만으로 배치 작업 가능
### 엑세스 관리 및 보안
> Amazon S3는 버킷 및 객체에 댛나 엑세스 감사 및 관리기능을 제공 기본적으로 프라이빗이나 생성한 S3 리소스에서만 엑세스 할 수있음
- **S3 퍼블릭 엑세스 차단** - 기본적으로 S3는 퍼블릭 엑세스가 차단되어 있으나 특정 사례가 있다면 허용하나 웬만하면 모든 퍼블릭 엑세스 차단 설정을 활성화 유지 하여야한다.
- **AWS Identity And Access Management** - IAM은 Amazon S3 리소스 포함 AWS 리소스에 대한 안정적인 엑세스를 제공  IAM의 세분 권한을 통해 S3 접근을 유지 할 수 있음
### 데이터 처리
- **S3 Object Lambda** - 자체 코드를 S3 GET, HEAD 및 LIST 요청에 추가하여 애플리케이션에 데이터가 변환 될 때 데이털르 수정 및 처리 가능
- **Event Alert**  - S3 리소스가 변경되면 Amazon SNS, SQS 및 Lambda 를 사용하는 workflow 트리거 합니다.

### 강력한 일관성
- Amazon S3는 모든 리전의 Amazon S3 버킷에 있는 객체의 PUT 및 DELETE 요청에 대해 강력한 쓰기 후 읽기(read-after-write) 일관성을 제공 새 객체에 대한 쓰기와 기존 객체 및 DELETE 요청을 덮어 쓰는 PUT 요청에 모두에 적용
- Amazon S3 객체 태그, 객체 메타데이터에 대한 읽기 작업은 매우 일관적임.


## 3. Amazon EFS (Elastic File System)
---
### 탄력적인 고가용성 파일 데이터 공유 서비스
- 완전히 탄력적인 서버리스 파일 스토리지 제공
- 스토리지 용량과 성능을 프로비저닝 하지 않아도 파일 데이터 공유가능
### 파일 시스템 유형
- **리전** - 리전 파일 시스템은 동일한 AWS 리전 내에서 여러 가용영역에 데이터  중복 저장
- **One Zone** - 단일 가용 영역 내에 데이터를 저장  가용영역 장애시 데이터가 손실

## 4. Amazon Glacier
---
### Valut 를 이용한 데이터 보관 및 장기 백업을 위한 안정적이나 저렴한 스토리지 서비스
- Amazon S3 Glacier 스토리지 클래스와 구별되는 자체 API를 갖춘 독립 실행 서비스
- 무기한으로 안전하게 보관되며 엑세스 가능 마이그레이션 필요하지 않으며 AWS 에서는 Stroage class로 옮기는 것을 권장
### 스토리지 클래스
- **Amazon S3 Glacier Instant Retrieval** - 분기당 한번 엑세스되고 밀리초 단위의 빠른 속도 검색이 필요한 수명이 긴 데이터에 보관 S3 Standdart-Infrequent Access 보다 최대 64% 가격이 저렴
- **Amazon S3 Glacier Flexible Retrieval** - 엑세스 시간과 대량 검색의 비용 균형을 유지하는 검색옵션을 제공 한다.
- **Amazon S3 Glacier Deep Arhchive** -  최저 비용의 스토리지 연 1회 미만으로 엑세스 되고 비동기식으로 검색되는 수명이 긴 아카이브 데이터 저장

## 5. AWS Backup
---
### 서비스,클라우드 및 온프레미스 에서 데이터 보호를 쉽게 중앙 집중화하며 자동화 할 수있는 완전 관리형 AWS 서비스
- AWS 리소스에 대한 백업 정책을 구성 활동을 모니터링 
- 서비스 별로 백업 작업을 자동화 통합이 가능 
- 사용자 지정 스크립트 나 수동 작업하지 않아도 된다. 
### 기능
- **중앙 집중식 백업 관리** - 애플리케이션이 사용하는 AWS 서비스 전체에서 백업을 관리하는 중앙 집중식 백업 콘솔 , 백업 APis 세트 및 AWS CLI 를 제공 리소스에 적용하여 일관되고 규정을 주사하는 방식으로 백업할 수 있음
- **정책 기반 백업 ** - Backup Plan 을 사용하면 조직 및 애플리케이션 전반에 걸쳐 확장 가능한 백업전략 쉽게 적용 가능
- **태그 기반 백업정책** - AWS Backup를 사용하여 리소스에 태그 지정 AWS 리소스에 백업 계획 적용이 가능하다.
- **Cross Region Backup** - 백업을 여러 다른 AWS 리전 온디맨드로 복사 예약된 백업 계획으 ㅣ일부로 자동 복사할 AWS 백업할 수 있다. 비즈니스 연속성 가용성을 부가할 수 있음
